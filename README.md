# 오디약 (WHERE IS THE PHARMACY?)
> <br/>
> 오디약은 서울에 거주하는 모든 분들이 약국을 쉽게 찾을 수 있도록 도와주는 서비스입니다. 
>
> 오디약을 이용하면 현재 영업 중인 약국, 야간 영업 중인 약국, 공휴일에 영업하는 약국, 내 위치 기반 약국을 찾을 수 있습니다.
>
> 또한, 외국어로 의사소통할 수 있는 약국을 제공합니다.
>
> 오디약은 외국인 대상 설문조사 결과를 바탕으로 개발되었지만, 한국인분들도 빠르고 쉽게 약국 위치를 찾을 수 있도록 도와줍니다. 
>
> 오디약은 서울에서 약국을 찾으시는 모든 분들이 현재 영업중인 약국을 신속하게 조회 가능한 보건 의료 서비스 입니다. <br/>

## 💡 기술적 의사결정

**FE**

| 사용 기술 | 도입이유 | 의견결정 |
| --- | --- | --- |
| React-query | 서버와의 데이터 통신을 통해 받은 데이터를 간편하게 관리하기 위함. |  캐시와 자동 재요청기능 및 캐시무효화기능을 통해 쉽게 데이터를 관리할 수 있고 로딩, 에러상태, 데이터 상태등을 직관적으로 다룰 수 있는 장점이 있으며, 추후 최적화에 사용할 페이지네이션 기능도 쿼리에 내장이 되어있어 선정 |
| Portal | 하나의 루트에 모달창을 팝업하는건 진정한 의미의 모달이 아니라 판단 | 모달을 전용으로 팝업시키기위한 전용 DOM트리 개설의 필요성을 느껴 이를 가능하게끔하는 portal을 사용 각각의 모달창의 컴포넌트를 포탈을 이용해 전용 DOM트리로 이동시키위해 재사용 portal 파일 작성 |
| Pagination | 검색 시 데이터를 불러오는 소요시간을 단축하고 싶었으며, 지도에 약국 핀이 많이 찍혀서 겹치는 걸방지하고자 지도에 찍히는 약국 개수를 제한하고싶었음. | 페이지 번호나 ">" 꺽쇠와 같은 아이콘을 클릭하여 간단하게 페이지를 이동할 수 있으며 직관적인 인터페이스를 제공해서 react-js-pagination 라이브러리를 사용 했습니다. |
| recoil | 약국 조건검색 후 상세페이지에 갔다가 다시 약국조회페이지에 돌아왔을때 조건값이 초기화가 되는 이슈로 인해 이전 조건값을 기억하기위해 전역상태관리가 필요하다고 판단  | Redux Toolkit은 redux보다 보일러플레이트가 적은 상태 관리 라이브러리이기도 했지만, 저는 더 간단한 구성과 사용성을 가진 Recoil로 결정했습니다. Recoil은 복잡한 상태 구조를 관리하거나 재사용 가능한 로직을 작성하는 데 유리하며, 추후에 타입스크립트를 도입할 예정이기 때문에 Recoil이 타입스크립트를 완벽하게 지원하는 점도 고려하여 선택하였습니다. |
| styled-components | 스타일을 인라인 또는 CSS파일로 하기보단 현재 추세에 맞는 라이브러리를 사용하기로함 | 이중에 styled-components를 사용한 이유는 props로 내려서 스타일을 지정을 해줄 수 있고 현재는 기간에 맞춰 구현을 완료하자를 목표로 프로젝트방향성을 잡아서 러닝커브 있는 테일윈드보다 상대적으로 익숙한 스타일드컴포넌트를 사용했습니다. |
| js-cookie | 서버로부터 받아오는 jwt 토큰의 관리를 위해 cookie에 저장하기로 결정 | 쿠키를 관리하기위한 라이브러리 중 react-cookie , js-cookie 중 js-cookie를 선택 aixos instance를 전응으로 관리하기위한 리액트함수형 컴포넌트가 아닌 js파일에서 사용하기에 react-cookie는 불편함이 있다고 판단함 react-cookie에서 사용하는 useCookies는 리액트훅이기에 리액트함수 외부에서 사용하기에 제약사항이 있어 js-cookie라이브러리를 선택 |
